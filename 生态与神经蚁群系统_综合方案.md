## 一、总览

完整的生态场景系统由以下子系统构成：

```

  ├─ 物种注册表 (SpeciesRegistry)    10蚂蚁 · 12植物 · 6+昆虫
  ├─ 植物群落 (Plant Community)      12种真实植物 + 草籽散播

  ├─ 属性面板 (Property Panel)        第二页科普·第三页代码模板
  ├─ 时间系统 (Time System v3.0)     饱食度·寿命·60fps基准

```

### 1.1 60fps参数校准基准（v3.0 时间系统重做）

所有生物参数按 **60fps游戏基准** 校准，重点解决以下问题：
- **蚂蚁饿死太快**：从每秒-1改为每10秒-1（17分钟饿死）
- **攻击节奏过快**：从每0.5秒1次改为每2秒1次
- **移动速度过快**：统一校准到 0.5-1.0 px/帧
- **寿命系统缺失**：添加自然寿命机制 60-120分钟

| 指标 | 建议范围 | 说明 |
|------|---------|------|
| **蚂蚁** 移动速度 | 0.5-1.0 px/帧 | ≈30-60px/秒，横穿800px需13-27秒 |
| **昆虫** 移动速度 | 0.5-1.5 px/帧 | 飞行昆虫略快但可观察 |
| **攻击**频率 | 每120帧(2秒)1次 | 战斗节奏舒缓，可观察 |
| **单次**攻击力 | 1-5伤害 | 工蚁低，兵蚁高 |
| **HP** | 25-60 | 15-30次攻击击杀一只 |
| **饱食度消耗** | 每600帧(10秒)-1 | 100饱食≈17分钟饿死 |
| **植物**生长 | 每600帧(10秒)+0.03 | 约5分钟长大 |
| **草籽**散布 | 每1200帧(20秒)1粒 | 食物资源稳定生成 |
| **自然寿命** | 216,000-432,000帧 | 60-120分钟游戏时间 |

### 1.2 真实蚂蚁与游戏时间的类比

| 维度 | 真实蚂蚁（工蚁） | 游戏等效 |
|------|------------------|---------|
| 寿命 | 7-70天（取决于蚁种和季节） | 60-120分钟（可观察完整生命周期） |
| 觅食频率 | 每天外出觅食多次 | 每17分钟需觅食补充饱食度 |
| 战斗 | 遇到敌蚁/敌虫时战斗 | 每2秒攻击一次，战斗可观察 |
| 搬运重量 | 自身体重10-50倍 | 草籽能量8-15单位 |

**核心设计原则**：游戏时间 ≈ 真实时间的 10-20倍加速，保证玩家在合理游戏时长内能观察到出生→觅食→繁殖→死亡的完整生命周期。

## 四、地图预设系统（v4.0 新增）

### 4.1 设计理念

地图预设不是简单的"皮肤"，而是**完整的生态群落配置**：
- 不同地图有不同的背景颜色（模拟环境）
- 不同地图有不同的物种组合（模拟当地生物区系）
- 不同地图有不同的食物密度与敌对生物密度
- 某些物种是特定环境**特有**的（如蚁狮只在沙漠/戈壁）

**关于"大树"的说明**：
不模拟具体树种的3D/复杂形态（技术上不可行且偏离核心游戏玩法），但可以用**植物物种的"大个体"**来表现：在落叶阔叶林/雨林地图中放置更大尺寸的植物基圆代表"树"，并配上对应的科普描述（如蒙古栎、榕树等）。

## 六、能量系统与碰撞检测

### 6.1 现有功能复用

引擎已实现以下能量相关功能：

**碰撞检测** (`cellCore.js`):
- `checkCollisions()` - 每帧检测基圆碰撞
- 碰撞时触发 `onCollision` 事件
- 防止基圆重叠，自动分离过近的基圆

**基圆销毁** (`sandbox.js`):
- `api.destroyCell(targetId)` - 删除指定基圆
- 能量为0时可调用此函数删除

### 6.2 能量吸取代码示例

```javascript
// 蚂蚁行为代码中的能量吸取逻辑
api.on('onCollision', (data) => {
  const other = CellCore.getCell(data.otherId);
  if (!other) return;
  const attrs = other.attributes || {};

  // 吸取草籽能量
  if (attrs.seedEnergy > 0) {
    const energy = api.getProperty('energy') || 0;
    const gained = Math.min(attrs.seedEnergy, 30);
    api.setProperty('energy', energy + gained);

    // 草籽能量耗尽后删除
    attrs.seedEnergy -= gained;
    if (attrs.seedEnergy <= 0) {
      api.destroyCell(data.otherId);
    }
  }

  // 吸取昆虫能量
  if (attrs.energyValue > 0 && attrs.hostile) {
    const energy = api.getProperty('energy') || 0;
    api.setProperty('energy', energy + attrs.energyValue * 0.5);
  }
});

// 能量耗尽自动删除
api.on('onUpdate', () => {
  const energy = api.getProperty('energy') || 0;
  if (energy <= 0) {
    api.destroyCell(api.getProperty('id'));
  }
});
```

## 七、神经蚁群系统

### 6.1 核心理念

"神经元"不是独立基圆类型，而是**计算范式**——把蚂蚁大脑编码为JSON基因数据，运行于行为代码内部。

```
蚁群管理器
  └─ 蚂蚁个体
       └─ 神经网络大脑（基因编码）
            ├─ 感觉神经元（传感器输入）
            ├─ 隐藏神经元（信号处理）
            └─ 运动神经元（行为输出）
```

### 6.2 基因组结构

```javascript
{
  "generation": 1,           // 代数
  "neurons": [
    { "id": 0, "type": "sensor", "label": "food_near" },
    { "id": 1, "type": "sensor", "label": "pheromone_strong" },
    { "id": 2, "type": "sensor", "label": "energy_level" },
    { "id": 3, "type": "hidden", "activation": "sigmoid" },
    { "id": 4, "type": "motor", "label": "move_direction" },
    { "id": 5, "type": "motor", "label": "move_speed" }
  ],
  "connections": [
    { "from": 0, "to": 3, "weight": 0.5 },
    { "from": 3, "to": 4, "weight": 0.8 }
  ],
  "params": { "mutationRate": 0.08, "mutationStrength": 0.2 }
}
```

### 6.3 行为代码模板

```javascript
api.setTriggerMode('continuous');

const genome = JSON.parse(api.getProperty('genome') || '{}');
let energy = api.getProperty('energy') ?? 80;
let frameCount = 0;

// 传感器读取
const nearby = api.queryCells({ kind: 'plant', near: { x: api.getX(), y: api.getY(), radius: 200 } });
const nearestFood = nearby[0];
const foodDist = nearestFood ? Math.hypot(nearestFood.x - api.getX(), nearestFood.y - api.getY()) / 200 : 1;

// 移动向食物
const speed = api.getProperty('speed') || 1.0;
if (nearestFood) {
  const dx = nearestFood.x - api.getX();
  const dy = nearestFood.y - api.getY();
  const dist = Math.hypot(dx, dy) || 1;
  api.setPosition(api.getX() + dx/dist * speed, api.getY() + dy/dist * speed);
} else {
  // 随机漫游
  api.setPosition(api.getX() + (Math.random()-0.5) * speed, api.getY() + (Math.random()-0.5) * speed);
}

// 能量消耗
if (frameCount % 60 === 0) {
```

### 8.3 能量/饱食度系统论证（最大改进）

**旧系统问题**：饱食度每60帧(1秒)-1，初始50 → **50秒饿死** → 完全不合理

**新系统（v3.0）**：饱食度每600帧(10秒)-1，初始100 → **1000秒≈17分钟饿死**

| 场景 | 饱食度变化 | 时间 | 说明 |
|------|-----------|------|------|
| 不觅食饿死 | 100 → 0 | 17分钟 | 合理：给蚂蚁充分觅食时间 |
| 吃到草籽恢复 | +15 | 瞬间 | 找到一颗草籽可延长约2.5分钟寿命 |
| 自然寿命结束 | age > maxAge | 60-120分钟 | 类比真实蚂蚁7-70天寿命 |

**与真实蚂蚁的类比**：

```
真实工蚁寿命：7-70天
游戏蚂蚁寿命：60-120分钟
时间比例 = 真实1天 ≈ 游戏1-3分钟
         = 游戏1分钟 ≈ 真实8-24小时
```

这个比例保证了在一次游戏会话（30-60分钟）内可以观察到：
- 蚂蚁出生 → 觅食 → 返巢 → 繁殖（完整生命周期）
- 多个世代的更替（2-4代）
- 种群数量变化

**平衡验证**：
- 1株植物每5分钟产15粒草籽 = 每5分钟提供120-225能量
- 1只蚂蚁每17分钟消耗100能量 = 每5分钟消耗约30能量
- 结论：1株植物可养活4-7只蚂蚁 → 合理的生态平衡
